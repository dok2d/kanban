# Kanban — персональная доска задач

[English version](README.md)

Минималистичный self-hosted канбан с эпиками, тегами, комментариями, уведомлениями, интеграцией с Telegram и drag-and-drop.

## Стек

- **Go 1.22** — stdlib `net/http` для сервера, шаблонов и роутинга
- **SQLite** (единственная внешняя зависимость через `mattn/go-sqlite3`)
- **Vanilla JS** — никаких фреймворков на фронте
- **Podman** — rootless контейнер с hardening

## Возможности

- Канбан-доска с перетаскиванием карточек между колонками
- Создание / редактирование / удаление задач с инлайн-редактированием
- Приоритеты (без, низкий, средний, высокий, критический)
- Опциональный дедлайн с визуальным индикатором на карточках
- Эпики с цветовой маркировкой и прогрессом
- Теги (множественные на задачу)
- Вложенные комментарии с ответами (Markdown, @упоминания)
- Зависимости между задачами с поиском/фильтрацией (зависит от / блокирует)
- Markdown-описания с подсветкой синтаксиса
- Интерактивные TODO-списки
- Вложения файлов и вставка изображений (Ctrl+V)
- Поиск по задачам, комментариям, тегам и эпикам (текст / regex)
- Управление колонками (создание, сортировка, удаление)
- Три роли: Администратор, Пользователь, Только чтение
- Система уведомлений (в приложении + Telegram)
- Подписки на задачи для отслеживания обновлений
- Интеграция с Telegram-ботом для push-уведомлений
- Лента активности пользователей с детальной историей изменений
- 8 тем оформления (тёмная, светлая, океан, лес, nord, dracula, solarized, spacedust)
- 10 языков (русский, английский, китайский, испанский, французский, немецкий, португальский, японский, корейский, арабский)
- Настраиваемый размер шрифта
- Выбор часового пояса
- Экспорт / импорт доски в JSON
- Все статические ресурсы локальные (без внешних CDN)
- Адаптивный дизайн для мобильных устройств

## Быстрый старт

```bash
# Сборка
./kanban.sh build

# Запуск
./kanban.sh run

# Открыть http://127.0.0.1:8080
```

При первом запуске будет предложено создать учётную запись администратора.

## Команды

| Команда               | Описание                              |
|-----------------------|---------------------------------------|
| `./kanban.sh build`   | Собрать образ контейнера              |
| `./kanban.sh run`     | Запустить контейнер                   |
| `./kanban.sh stop`    | Остановить                            |
| `./kanban.sh restart` | Перезапустить                         |
| `./kanban.sh logs`    | Логи                                  |
| `./kanban.sh backup`  | Бэкап БД в ./backups/                 |
| `./kanban.sh status`  | Статус контейнера                     |
| `./kanban.sh deploy`  | Установить systemd + nginx конфиг     |

### Флаги (для `run` и `deploy`)

| Флаг              | Описание                              | По умолчанию          |
|-------------------|---------------------------------------|-----------------------|
| `--host <значение>` | FQDN или IP-адрес                  | `kanban.local`        |
| `--port <порт>`   | Порт контейнера                      | `8080`                |
| `--tls`           | Включить TLS (HTTPS)                 | включён               |
| `--no-tls`        | Только HTTP, без TLS                 | —                     |
| `--cert <путь>`   | Путь к TLS-сертификату               | `/etc/nginx/ssl/kanban.crt` |
| `--key  <путь>`   | Путь к TLS-ключу                     | `/etc/nginx/ssl/kanban.key` |

Флаги дублируются переменными окружения: `KANBAN_HOST`, `KANBAN_PORT`, `KANBAN_TLS`, `KANBAN_SSL_CERT`, `KANBAN_SSL_KEY`.

```bash
# Запуск локально на другом порту
./kanban.sh run --port 9090

# Деплой с TLS
./kanban.sh deploy --host kanban.example.com --port 9090

# Деплой без TLS (только HTTP)
./kanban.sh deploy --host 10.0.0.5 --port 8080 --no-tls
```

## Авторизация и роли

Приложение требует авторизации. При первом запуске необходимо создать учётную запись администратора через страницу настройки.

Три роли:
- **Администратор** — полный доступ: управление пользователями, колонками, эпиками, тегами, настройками Telegram-бота
- **Пользователь** — создание/редактирование/удаление задач, комментирование, подписка на уведомления
- **Только чтение** — просмотр доски и задач, получение уведомлений

Сессии основаны на cookie (90 дней), пароли хешируются PBKDF2-HMAC-SHA256.

Восстановление пароля доступно для пользователей с привязанным Telegram — 6-значный код отправляется в бот.

## Интеграция с Telegram

Администраторы могут настроить Telegram-бота для push-уведомлений:

1. Создать бота через [@BotFather](https://t.me/BotFather) и получить токен
2. В Настройках → Telegram ввести токен бота и имя бота
3. Пользователи привязывают аккаунт, отправив хеш-код боту

Уведомления отправляются при: назначении задачи, @упоминаниях, комментариях к отслеживаемым задачам и обновлениях задач.

Команды бота:
- `/tasks` — список ваших задач
- `/task N` — просмотр задачи #N с последними комментариями
- `/comment N текст` — оставить комментарий к задаче #N
- `/help` — список команд

## Безопасность контейнера

- **Non-root**: процесс запускается от пользователя `kanban` (не root)
- **Read-only filesystem**: корневая ФС монтируется только на чтение
- **CAP_DROP ALL**: все capabilities сброшены
- **no-new-privileges**: запрет эскалации привилегий
- **Лимиты**: 256MB RAM, 0.5 CPU
- **Слушает только 127.0.0.1**: наружу только через nginx

## Деплой (Systemd + Nginx)

Команда `deploy` генерирует и устанавливает systemd quadlet unit-файлы
и nginx reverse proxy конфиг за один шаг:

```bash
# TLS (по умолчанию) — HTTPS nginx конфиг + systemd units
./kanban.sh deploy --host kanban.example.com --port 9090

# Только HTTP
./kanban.sh deploy --host 10.0.0.5 --port 8080 --no-tls

# Затем запустить
systemctl --user daemon-reload
systemctl --user start kanban
systemctl --user enable kanban
sudo nginx -t && sudo systemctl reload nginx
```

Self-signed сертификат:
```bash
sudo mkdir -p /etc/nginx/ssl
sudo openssl req -x509 -nodes -days 3650 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/kanban.key \
  -out /etc/nginx/ssl/kanban.crt \
  -subj "/CN=kanban.example.com"
```

## API

Все эндпоинты возвращают JSON. Требуется авторизация (session cookie).

### Авторизация

| Метод  | Путь              | Описание                         |
|--------|-----------------  |----------------------------------|
| POST   | /api/auth/setup   | Создание первого администратора  |
| POST   | /api/auth/login   | Вход                             |
| POST   | /api/auth/logout  | Выход                            |
| GET    | /api/auth/me      | Информация о текущем пользователе|
| POST   | /api/auth/reset-request | Запрос сброса пароля (через Telegram) |
| POST   | /api/auth/reset-confirm | Подтверждение сброса кодом       |
